# NerdMadness Multi-Bracket System Plan

**Branch**: `feature/nerds-play-basketball-too`
**Created**: 2026-02-24
**Status**: PLANNING

---

## 1. Executive Summary

Convert NerdMadness from 1:1 (one bracket per user) to 1:N (multiple brackets per user) with admin management capabilities.

### Key Requirements
| Requirement | Detail |
|-------------|--------|
| **Relationship** | One user → Many brackets |
| **Naming Flow** | Complete all 63 picks → THEN name bracket |
| **Edit Capability** | Rename allowed before lock |
| **Lock Time** | March 19, 2026 at noon EST |
| **Active Toggle** | Binary flag for soft delete |
| **Inactive Visibility** | Greyed before lock → Hidden after lock |
| **Admin Access** | 2 UIDs only |
| **Reactivation** | Admins only, before lock time |

---

## 2. Data Model Changes

### 2.1 Current Structure (1:1)
```
Path: artifacts/nerdbasketball/pools/nerdmadness_2026/brackets/{userId}
```
- User ID IS the document ID
- Only one bracket per user possible

### 2.2 New Structure (1:N)
```
Path: artifacts/nerdbasketball/pools/nerdmadness_2026/brackets/{autoGeneratedId}
```

**Bracket Document Schema:**
```javascript
{
  bracketId: string,              // Auto-generated Firestore ID
  userId: string,                 // Reference to user
  bracketName: string | null,     // Set LAST after all picks (null until named)
  picks: {                        // 63 game picks
    game1: { winner: "teamId", ... },
    // ... game63
  },
  tiebreakerTotal: number,        // Championship total score prediction
  active: boolean,                // true = active, false = soft deleted
  isComplete: boolean,            // true when all 63 picks made
  createdAt: Timestamp,           // When bracket created
  updatedAt: Timestamp,           // Last modification
  submittedAt: Timestamp | null   // When finalized with name (null until submitted)
}
```

### 2.3 Settings Metadata Enhancement
```
Path: artifacts/nerdbasketball/pools/nerdmadness_2026/metadata/settings
```

**Add/Update Fields:**
```javascript
{
  // Existing fields...
  lockTime: Timestamp("2026-03-19T12:00:00-05:00"),  // NEW: Tournament lock
  tournamentName: "NerdMadness 2026"                  // Existing
}
```

## 5. User Flow Diagrams

### 5.1 Create New Bracket
```
User lands on nerdmadness.html
    ↓
[+ Create New Bracket] button
    ↓
Redirected to bracket editor (blank)
    ↓
User makes 63 picks
    ↓
All picks complete → Prompt for bracket name
    ↓
User enters name → Save bracket
    ↓
Redirected to "My Brackets" list
```

### 5.2 Edit Existing Bracket (Before Lock)
```
User views "My Brackets" list
    ↓
Clicks [Edit] on a bracket
    ↓
Bracket editor loads with existing picks
    ↓
User modifies picks/name
    ↓
Save changes
    ↓
Return to "My Brackets"
```

### 5.3 After Lock (March 19, 2026 noon EST)
```
User views "My Brackets" list
    ↓
All [Edit] buttons disabled
    ↓
Inactive brackets HIDDEN (not just greyed)
    ↓
Can only VIEW active brackets
    ↓
No modifications possible
```

### 5.4 Admin Flow
```
Admin navigates to nerdmadness-admin.html
    ↓
Auth check (UID in ADMIN_UIDS?)
    ↓
YES → Show all brackets table
    ↓
Admin can:
  - View any bracket details
  - Toggle active/inactive (deactivate anytime)
  - Reactivate (ONLY before lockTime)
  - Filter/search brackets
```
